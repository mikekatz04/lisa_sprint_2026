# Sprint-wide pip constraint file.
#
# Why: pybind11 keys its process-global type registry by PYBIND11_INTERNALS_ID,
# a string baked in at compile time from (pybind11 version, compiler ABI tag,
# PYBIND11_BUILD_ABI). If GPUBackendTools, LISAanalysistools, GBGPU, BBHx, and
# FastEMRIWaveforms are each built against a different pybind11, their
# internals dicts diverge -- a class registered by LAT becomes opaque to GBGPU
# at cross-wheel cast time, silently. This file pins the version sprint-wide
# so that all wheels emit the same internals ID.
#
# How to use:
#
#   export PIP_CONSTRAINT="$(git rev-parse --show-toplevel)/constraints/sprint.txt"
#   pip install -e GPUBackendTools/
#   pip install -e LISAanalysistools/
#   pip install -e GBGPU/   # etc.
#
# Every CI workflow in the sprint must also set PIP_CONSTRAINT before any
# pip install step.  See plan §"Sprint-wide pybind11 pin" for rationale.
#
# Bump the pin in ONE PR sprint-wide; per-package upgrades are forbidden.

pybind11==3.0.4
